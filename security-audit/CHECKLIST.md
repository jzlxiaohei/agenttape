# AgentTape 开源前安全检查关键项

## 审计边界

AgentTape 是用户在自己电脑上运行的本地工具，不按公网多租户 SaaS 的威胁模型审计。重点保护：用户本机、API 凭证、捕获的 Trace、配置文件，以及从源码到发布物的供应链。

## P0：开源前必须通过

### 1. 仓库与发布物不泄密

- 当前工作树、Git 历史、测试数据和文档中没有真实 API Key、Token、Cookie、账号或私有证书。
- `.gitignore`、`.dockerignore` 和发布文件白名单排除 `.env`、本地数据库、原始 Trace、日志、备份、缓存及构建临时文件。
- 示例配置仅使用明显无效的占位值；错误信息和日志不输出凭证。

### 2. 本地 HTTP 服务边界安全

- 默认仅监听 loopback（`127.0.0.1`/`::1`）；绑定非 loopback 地址需要明确警告或显式危险开关。
- 有副作用的接口不能被任意网页通过 CSRF、简单请求或宽松 CORS 调用。
- 校验 `Origin`/`Host`，防止恶意网页、DNS rebinding 或伪造 Host 访问 localhost 能力。
- 浏览器安全响应头合理，包括 CSP、`X-Content-Type-Options`、frame 限制和 Referrer Policy。
- 启动进程、重放请求、删除数据、修改本机配置等高风险能力有独立开关且默认安全。

### 3. 凭证与 Trace 数据安全

- `Authorization`、API Key、Cookie 等认证头不落盘，或在落盘前可靠脱敏。
- UI、搜索、导出、错误响应和日志不会意外展示凭证。
- 文档明确说明 Prompt、响应、工具调用和文件内容可能被完整捕获并保存在本机。
- 数据目录和敏感文件使用最小权限；临时凭证及时清理。

### 4. 不可信 Trace 与输入安全

- Markdown、代码、HTML、URL 和富文本按不可信内容处理，不能执行脚本或危险协议。
- 请求体、响应体、Hook、导入文件和 JSON 解码均有合理大小限制。
- 文件名、路径、URL、正则和数据库查询参数经过约束，不能路径穿越、SQL/命令注入或造成明显资源耗尽。
- 上游地址、重放目标和代理路径不能被越权利用来访问本机/内网，或此能力必须明确标为高风险并受保护。

### 5. 本机执行与配置修改安全

- 如果产品明确允许本机用户输入 Shell 片段，应把它标为受信任的高级能力，并确保只有受信任的本地 UI/调用方能够触发；否则使用结构化参数和转义。
- 工作目录、可执行文件、终端应用、命令参数和 Hook 配置不能被远程网页任意控制。
- 修改用户配置前可靠备份；并发、崩溃和重复启动时可恢复，不覆盖用户新修改。
- 临时脚本/文件权限最小，使用后删除，不把 API Key 写入其中。

## P1：发布前应通过

### 6. 依赖与构建供应链

- Go 与前端锁文件完整，依赖无已知高危漏洞。
- 审查安装/构建脚本、Git 依赖、废弃依赖和异常来源。
- CI 对来自 Fork 的代码不暴露发布密钥，工作流权限最小且关键 Action 固定版本。
- 发布物可追溯到源码，生成过程可复现或至少有校验和。

### 7. 数据库与文件生命周期

- SQL 使用参数绑定；删除、导出和原始文件读取校验资源归属与路径。
- 数据目录、数据库、原始 Trace、备份和临时文件的权限及清理策略明确。
- 避免符号链接、TOCTOU 和任意文件读取/覆盖风险。

### 8. 安全默认值与文档

- README 明确：仅本地使用、不要直接暴露公网、Trace 可能包含敏感数据。
- 危险功能默认关闭，或首次使用前有清晰确认和风险说明。
- 提供 `SECURITY.md`，说明支持版本、私下报告渠道及响应预期。
- 记录已知限制和用户必须承担的信任边界。

## P2：持续改进

- CI 中加入秘密扫描、依赖漏洞扫描、静态分析和基础安全测试。
- 为 Origin/Host、CORS/CSRF、XSS、请求大小、路径处理和敏感头脱敏增加回归测试。
- 发布前复查新增接口、依赖、外部请求和本机能力。
- 对最终二进制、前端构建目录和发布压缩包再次扫描，而不只扫描源码。

## 不纳入本轮的 SaaS 项目

- 多租户隔离、注册登录、密码找回和企业组织权限。
- WAF、大规模 DDoS、云网络隔离、多区域灾备。
- 公网数据库、负载均衡和 Kubernetes 集群加固。

如果未来支持局域网共享、远程访问或托管部署，以上项目必须重新纳入威胁模型。
